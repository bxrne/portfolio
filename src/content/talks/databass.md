I introduced Tau, a bitemporal database written in Rust, designed to handle time-travel queries and maintain historical data efficiently. The project focused on implementing a robust data model that supports both valid time and transaction time, allowing users to query the state of the database at any point in history. With QoL features like a TCP QL interface, Materialised and non Materialised transformations and advanced correction handling. Hosted by Alex Petrov.
